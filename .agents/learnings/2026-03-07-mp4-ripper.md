# Learning: yt-dlp Silently Embeds Subtitles During Merge

__ID__: L1
__Category__: architecture
__Confidence__: high

## What We Learned

When `writesubtitles=True` is set alongside `merge_output_format` in yt-dlp, the merge pass silently absorbs any downloaded subtitle files into the MP4 container as soft subtitle tracks. The standalone `.srt` file is never written to disk. The only reliable way to produce a separate `.srt` file is a second, dedicated `skip_download=True` pass with subtitle flags isolated from the video download.

## Why It Matters

Any yt-dlp integration that needs external subtitle files (for SRT delivery or pre-burn inspection) must decouple the subtitle fetch from the video/audio download. Combining them is a silent failure mode — no error is raised, the subtitle just disappears into the container.

# Learning: YouTube Auto-Captions Require a Three-Pass SRT Cleaner

__ID__: L2
__Category__: debugging
__Confidence__: high

## What We Learned

YouTube auto-generated captions arrive in two distinct messy formats after yt-dlp conversion to SRT: (1) rolling-word prefix chains where each entry is a superset of the previous (e.g., "Hello" → "Hello world" → "Hello world now"), and (2) same-timestamp multi-line blocks where two entries share an identical start time. A single-pass prefix check handles only pattern 1. A complete cleaner needs three passes: prefix drop → same-timestamp merge → overlap trim.

## Why It Matters

Any project that burns or displays YouTube auto-captions must apply this cleaner or subtitles will visually stack on screen. Applies to any FFmpeg subtitle burn pipeline, not just this project.

## Source

mp4-ripper — burned caption mode, discovered when two-line stacking persisted after first fix attempt.

# Learning: FFmpeg `subtitles=` Filter Breaks on Windows Drive-Letter Colons

__ID__: L3
__Category__: debugging
__Confidence__: high

## What We Learned

FFmpeg's `subtitles=` video filter uses colons as a syntax delimiter. On Windows, any path containing a drive letter (e.g., `C:\path\to\file.srt`) causes a parse error because the `C:` is interpreted as a filter option separator. The cross-platform fix is to set `cwd=` in `subprocess.run` to the file's parent directory and pass only the bare filename to the filter — no path, no colon problem.

## Why It Matters

This is a silent cross-platform landmine. Code that works on macOS/Linux will fail on Windows without this workaround. Must be applied to any FFmpeg filter that accepts file paths as arguments.

## Source

mp4-ripper — `_burn_subtitles()` implementation, caught during architecture review before reaching the user.

# Learning: Startup Dependency Checks Beat Runtime Error Handling for CLI Tools

__ID__: L4
__Category__: architecture
__Confidence__: high

## What We Learned

For desktop apps that depend on external CLI tools (FFmpeg, yt-dlp, etc.), checking for the dependency at application startup and surfacing a clear, human-readable error immediately is far more valuable than catching subprocess errors at runtime. A missing FFmpeg at startup produces a modal dialog with install instructions; a missing FFmpeg mid-download produces a cryptic subprocess traceback.

## Why It Matters

Applies to any tool that shells out to external binaries. Startup checks are a one-time cost that eliminate an entire class of confusing runtime failures for users who haven't yet configured their environment.

## Source

mp4-ripper — `check_ffmpeg()` called in `__init__`, validated during QA.

# Learning: Windows 11 Media Player (v11) is Not Classic WMP 12

__ID__: L5
__Category__: debugging
__Confidence__: high

## What We Learned

The new Windows 11 Media Player app (v11.x, redesigned UX) is an entirely different application from the classic Windows Media Player 12 (legacy). Online documentation and support articles almost universally conflate the two. The right-click "Lyrics, Captions and Subtitles" menu exists only in WMP 12, not the new app. External SRT soft-subtitle support in the new app is unreliable. VLC is the correct cross-platform reference player for SRT testing and delivery.

## Why It Matters

Any documentation or user-facing guidance about subtitle playback on Windows should explicitly distinguish between the two applications and recommend VLC for soft-subtitle workflows.

mp4-ripper — SRT mode user testing on Windows 11 workstation.

# Learning: Tkinter Thread Safety Requires `after(0, ...)` for All Callback Updates

__ID__: L6
__Category__: architecture
__Confidence__: high

## What We Learned

Tkinter is not thread-safe. Any widget update (text, progress bar, button state) that originates from a background thread must be routed through `widget.after(0, lambda: ...)`. Direct widget writes from non-main threads cause silent corruption or crashes that are difficult to reproduce. This pattern must be applied to every progress callback, status update, and done/error handler.

## Why It Matters

Applies to every Python desktop app that uses Tkinter with background threads. The pattern is non-negotiable — there is no other safe way to update Tkinter state from a worker thread.

## Source

mp4-ripper — `download_in_thread` callback architecture, `_on_progress`, `_on_status_update`, `_on_done`.
